# WORKSPACE_GRAPH.txt — the reviewed workspace-graph acknowledgment file (bead fln-8mj).
#
# This file is the single reviewed source of truth for the crate map of plan §21 and
# the layering/structural laws of plan §3 (D3) and §5. `tools/structure-guard` compares
# the actual cargo workspace against this file on every CI run:
#
#   * every crate on disk must be declared here, and vice versa           (snapshot law)
#   * every inter-crate dependency edge must be declared here, and vice versa
#   * every edge must point strictly downward: rank(from) > rank(to)      (§5 layering)
#   * declared transitive prohibitions must hold over the real edge set   (D3 laws)
#   * kernel/checker direct dependencies must stay inside their allowlists (§8 covenant)
#   * external dependencies must be FrankenSuite members, by path          (D1 closure)
#
# Adding a crate or an edge REQUIRES editing this file in the same change — that edit
# is the review surface. Ranks encode the strictly-downward order of §5/§21; amending
# a rank is a reviewed architectural decision, not a convenience.
#
# Grammar (line-oriented; '#' starts a comment):
#   schema fln-workspace-graph/1
#   crate <name> rank=<n> kind=ordinary|unsafe-boundary     (product crates, crates/)
#   crate <name> kind=tool                                  (dev apparatus, tools/)
#   edge <from> -> <to>                                     (declared dependency edge)
#   prohibit <pat> ->* <pat>                                (no transitive path; '*' suffix globs)
#   allow-direct <crate> = <dep>[, <dep>...]                (exhaustive direct-dep allowlist)
#   covenant <crate> max-loc=<n>                            (line-count covenant, src/**/*.rs)
#   suite-dep <package>                                     (closed-universe external allowlist)
