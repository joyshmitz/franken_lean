# CLOSURE_ALLOWLIST.txt — the governed dependency closure (plan D1; bead franken_lean-xwf).
#
# Every package in Cargo.lock must have exactly one row here, and every row must match a
# Cargo.lock package — both directions enforced by `tools/structure-guard` on every run.
# A diff to this file IS the review artifact for a closure change; CI fails on any
# unlisted package, version/source/checksum mismatch, or stale row. External (registry)
# packages are prohibited outright: only workspace crates and SUITE.lock-pinned
# FrankenSuite path dependencies can ever appear. No serde, no tokio, no rocksdb, no
# LLVM, no cranelift, no gmp-sys, no external SAT solver — ever.
#
# Row grammar (one row per line; reason= captures the rest of the line):
#   package <name> version=<v> source=workspace|suite checksum=<hex|-> license=<spdx>
#           build-script=yes|no proc-macro=yes|no native-link=yes|no
#           unsafe-audit=<forbid|deny-ledgered|external> policy=runtime|build|dev|fuzz
#           owner=<owner> upgrade=<policy> reason=<free text>
#
# source=workspace rows carry checksum=- (path members have no registry checksum);
# source=suite rows carry the SUITE.lock commit as their provenance and checksum=-.
# build-script=no is verified against the crate tree (no build.rs may exist).
